Independent quality control plane
Pattern Pilot gives the builder a second model that reviews the same codebase from a clean QC boundary.

Codex builds inside the target project. Pattern Pilot reviews that work from the same repository context, the same Git diff, and the same governance rules, while keeping review state in its own database. The target project stays the source of truth for code and workflow; Pattern Pilot owns only QC memory.

Shared repo access Git-scoped diffs Independent reviewer model Pattern Pilot audit trail No target-project writes

Traditional QC vs Pattern Pilot

Traditional loop

  • The same development context often produces and validates the change.
  • Review history is scattered across chat, PR comments, terminal output, and memory.
  • Governance depends on whether the reviewer remembered the right project rules.
  • Metrics are hard to recover after the task is done.

Pattern Pilot loop

  • The builder writes code; an independent reviewer model evaluates the result.
  • Every run, round, finding, advisory, and event is stored in Pattern Pilot.
  • Project governance is loaded into the review bundle on purpose.
  • Pass rate, round count, findings, cost, and audit history stay queryable.

Truth boundaries

APP

Target project owns

  • Source code
  • Git branches
  • Tasks and workflow
  • Governance files
||
PP

Pattern Pilot owns

  • Review runs
  • Review rounds
  • Findings and advisories
  • Metrics and audit events

Integration surfaces

  • MCP: Codex calls Pattern Pilot without leaving the coding workspace.
  • API: The dashboard, onboarding, metrics, reviews, and advisories use FastAPI.
  • Postgres: Pattern Pilot stores its own QC history outside the target project.
  • Connectors: Project adapters read files, governance, dependencies, and context.

Verdict behavior

BLOCKING

Must be fixed. The builder applies changes and resubmits.

PASS_WITH_ADVISORIES

Accepted with notes that remain visible for future review.

PASS

Clean completion. The task exits the QC loop.

FAILED

Infrastructure blocked evaluation. This is not a code-quality failure.
